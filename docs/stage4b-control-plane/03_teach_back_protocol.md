# Teach-Back Protocol

Each implementation slice begins with a short chat teach-back:

- Concept: the technical idea being applied.
- Intended code change: the files or behavior expected to change.
- Expected behavior: what should be observable after the slice.
- Likely interview question: one question the operator must be able to answer.

Each slice ends with:

- Changed files.
- Tests run.
- Evidence id.
- What the operator must be able to explain without AI.
