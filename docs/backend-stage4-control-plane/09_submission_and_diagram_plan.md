# Submission And Diagram Plan

The final deliverable is prepared locally first, then moved into Google Docs for manual submission.

## Draft

- Source: `drafts/insighta-labs-stage4-design.md`
- Final destination: Google Docs
- Publish route: manual paste or conversion
- Sharing requirement: "Anyone with the link can view"
- Length requirement: 2-7 pages

## Required Document Structure

1. Requirements: functional and non-functional.
2. Architecture: high-level design with justified components.
3. Data flow: ingestion, query processing, retrieval, and response.
4. Design decisions: choices, alternatives, trade-offs, and requirement mapping.
5. Trade-offs and limitations: degradation modes and intentional simplifications.
6. Optional future evolution: real-time analytics and true natural-language query support.

## Diagram

- Source folder: `diagrams/source/`
- Export folder: `diagrams/exported/`
- Required final state: diagram embedded directly in Google Docs and a link to the original source included.

## Asset Rules

- Diagrams must support real architecture claims.
- Diagram labels should match the final design language.
- Do not use decorative diagrams.
- Do not claim production readiness beyond the brief's scale assumptions.
